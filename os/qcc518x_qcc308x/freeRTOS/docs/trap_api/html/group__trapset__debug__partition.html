debug_partition_result DebugPartitionErase ( void  )

Erase Debug Partition Debug Partition will be erased, the trap blocks for the duration of erase. This can take up to several seconds and application will be unresponsive during this time. Erase can't be started if Debug Partition Source Stream is open, this has to be closed first.

Result code indicating success or error reason.

Source StreamDebugPartitionSource ( void  )

Return Debug Partition Source Stream Source stream can be used for reading data stored on the Debug Partition. The data is read sequentially from start to end, StreamSize() can be used to check if there is more data to be read but does not give exact size of the data. Instead size of data on the Debug Partition can be ascertained with DebugPartitionInfo() trap. Once end of the data is reached, the stream has to be closed. It can be then re-opened to start reading the beginning.

Returns
Stream Source for reading data.
