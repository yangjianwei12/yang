◆ CcProtocol_ConfigureAsWakeupSource()

void CcProtocol_ConfigureAsWakeupSource ( void  )

#include <adk/src/domains/case_comms/cc_protocol/cc_protocol.h>

Allow the app to configure charger comms as a dormant wakeup source, if necessary.

The hardware automatically wakes the chip from dormant if there is a change on VCHG. Some devices use LED pads for the charger comms interface. This function should be used to configure the dormant module for such scenarios.

◆ CcProtocol_Disable()

bool CcProtocol_Disable ( void  )

#include <adk/src/domains/case_comms/cc_protocol/cc_protocol.h>

Disable case comms.

Returns
bool TRUE Case comms is disabled. FALSE Case comms is not disabled.
Note
This API is only supported for Scheme B transport configuration.

When disabled, the UART used by the scheme B transport is available to be reconfigured and used by the application.

Using this API will cause any data in the UART and stream to be discarded. Any case comms transmissions to this device whilst disabled are ignored.

◆ CcProtocol_Enable()

bool CcProtocol_Enable ( void  )

#include <adk/src/domains/case_comms/cc_protocol/cc_protocol.h>

Enable case comms.

Returns
bool TRUE Case comms is enabled and available for use. FALSE Case comms is not enabled.
Note
This API is only supported for Scheme B transport configuration.

This API will attempt to reacquire the UART and chargercomms UART stream for use by case comms. The application must ensure any use of the UART has been completely stopped and the UART stream released, or this API will fail.

cc_cid_t  cid,

uint16  len 
)

#include <adk/src/domains/case_comms/cc_protocol/cc_protocol.h>

Transmit a message over case comms, where a response message is expected.

Parameters
destDestination device.
cidCase comms channel on which to transmit.
midCase comms message ID of the packet.
dataPointer to the packet payload.
lenLength in bytes of the payload.
Returns
bool TRUE packet will be transmitted. FALSE packet can not be transmitted.
Note
Return value of TRUE only indicates the packet has been accepted for transmission. Client must await CcProtocol_TxStatus_fn callback for indication of success/fail receipt by destination.
The response message expected by using this API is a new message sent by the remote device, not the ACK/NAK received by clients via the CcProtocol_TxStatus_fn callback, which is a transmit status indication. Using this API instructs the case comms protocol to poll the remote device if required to receive the response message.
