Detailed Description

Header file for Bluetooth Low Energy advertising functionality.

Message handler functions for the advertising set state machine.

Header file for management of Bluetooth Low Energy advertising sets.

Internal defines used by the advertising manager.

Functions to set and retrieve the default advertising parameters to use.

Data types used by the LE advertising aggregator.

Header file for LE advertising manager aggregator set.

Header file for LE advertising manager aggregator group.

Header file for LE advertising manager aggregator, which is responsible for collecting individual advertising items and creating advertising sets.

Header file for LE advertising manager functionality associated with advertising item database.

Provides control for Bluetooth Low Energy (BLE) advertisements.

An aggregator group is a collection of afvertising items that have compatible advertising parameters.

An aggregator set is the advertising items that will be included in a single OTA advertising set.

Unless an individual advertising item overrides the advertising parameters
with its own GetItemParameters implementation it will use the default
parameters.

The application can set its own config for some of the parameters by setting a
config table at initialisation time and then selecting which row of the table
to use at runtime. For example it could use faster default parameters when
in pairing mode but otherwise use slower parameters to save power.

◆ le_adv_event_t

enum le_adv_event_t

#include <adk/src/domains/bt/le_advertising_manager/multi_set/le_advertising_manager_multi_set.h>

Types to specify the individual LE advertising events. 

   LEAM_EVENT_ADVERTISING_SET_SUSPENDED:
   When the advertising set for the client's advertising item gets suspended, this LE advertising event gets generated
   to notify the advertising suspended event to the advertising item. Client that owns the advertising item
   needs to implement the callback NotifyAdvertisingEvent(...) to handle the event.

   LEAM_EVENT_ADVERTISING_SET_RANDOM_ADDRESS_CHANGED:
   This LE advertising event gets generated to notify clients of changes in their advert set's random address.
   Client that owns the advertising item needs to implement the callback NotifyAdvertisingEvent(...) to handle the event.
