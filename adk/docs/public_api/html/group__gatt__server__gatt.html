◆ GattServerGatt_SetGattDbChanged()

void GattServerGatt_SetGattDbChanged ( void  )

#include <adk/src/domains/bt/gatt/servers/gatt_server_gatt/gatt_server_gatt.h>

Called when the local GATT DB has changed (eg. after software upgrade)

If GATT services have changed on the local device, (eg. after a software upgrade), then it is necessary to inform any interested bonded clients of this. Clients can use the client configuration of the Service Changed characteristic to register for indications of such changes. This API will send indications to any connected clients that have registered and are bonded.

◆ GattServerGatt_SetServerServicesChanged()

void GattServerGatt_SetServerServicesChanged ( gatt_cid_t  cid)

#include <adk/src/domains/bt/gatt/servers/gatt_server_gatt/gatt_server_gatt.h>

Called for all GATT connections.

This API is used in place of GattServerGatt_SetGattDbChanged operation since the former API is not working as expected. This will set client configuration of the Service Changed characteristic for given CID for a handset device.

This is an workaround fix for DFU from a old version to new version when GATT DB has been modified. Send notification to connected handset device by invoking appDeviceSetGattServerServicesChanged

Parameters
cidGiven CID for a connected device
