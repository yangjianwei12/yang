Field Documentation

◆ activate_timestamps

const unsigned activate_timestamps

If activate_timestamps is set, a user needs timestamp information propagated through the chain. active_timestamps flag is intended for users that do not need a specific ttp_delay. A default ttp_delay of 40ms is set if no other user requests a ttp_delay value. If another user requests a different ttp_delay, this ttp_delay will be set. The ttp_delay value can change dynamically.

◆ mandatory_task_period_us

const uint32* mandatory_task_period_us

mandatory_task_period_us defines the required task period for each user in microseconds. If the value is not written, the default 2000 us task period will be used. The user shouldn't change the task period after registration.

◆ num_of_mandatory_mics

uint8 num_of_mandatory_mics

Mic interface will connect to all mandatory microphones of all registered users in case a non-interruptible client is connnecting. With that a transition between different use case can be achieved without discontinuities.
