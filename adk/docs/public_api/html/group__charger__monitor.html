◆ Charger_CanEnterDormant()

bool Charger_CanEnterDormant ( void  )

#include <adk/src/domains/power/charger_monitor/charger_monitor.h>

/brief Check if we know that the system can enter dormant.

Dormant can only happen even if the charger is connected, but not charging. This function therefore checks the charger state to determine if charger has power.

The presence of power at the charger interface does not prevent the system entering dormant.

If the power status is not known, the function assumes that we cannot power off. An example would be when charger detection is still pending and the USB status is unknown.

Note
If charger support has not been included, then this function will return FALSE as the system behaviour is not known/detectable.
Returns
TRUE if the charger is not currently charging and optionally no VCHG voltage is detected.

◆ Charger_CanPowerOff()

bool Charger_CanPowerOff ( void  )

#include <adk/src/domains/power/charger_monitor/charger_monitor.h>

/brief Check if we know that the system can power off.

Power off can only happen when there is no voltage at the VCHG input, so this function checks the charger state to determine if charger has power.

If the power status is not known, the function assumes that we cannot power off. An example would be when charger detection is still pending and the USB status is unknown.

Note
If charger support has not been included, then this function will return FALSE as the system behaviour is not known/detectable.
Returns
TRUE if it is certain there is no external power
