Detailed Description

Header file for USB application framework.

USB Voice application interface - enumerates HID consume transport, HID datalink and USB Audio (voice calls with support for either narrowband or wideband) classes.

USB Voice application interface - enumerates HID consume transport, HID datalink and USB Audio (voice calls) classes.

Header file for UCQ string descriptor.

USB Source Audio application - enumerates HID consumer transport, HID datalink and USB Audio classes.

Default USB Audio application interface - enumerates only HID datalink class.

USB application for USB.org Compliance testing - enumerates USB HID consumer transport, HID datalink, USB Audio and Mass Storage classes.

USB CDC application interface - enumerates Communications Devices class with 2 bulk endpoints for COM port like communications.

USB Audio application interface - enumerates HID consume transport, HID datalink and USB Audio classes which support both headphone (for music playback) and headset (for voice calls) devices with separate audio function.

USB Audio application - enumerates HID consume transport, HID datalink and USB Audio classes which support audio & voice use cases in same audio function with common speaker for audio & voice use cases. If mic interface is active with or without speaker being active, then voice chain will be created. If speaker interface is active and mic interface is not active, then audio chain will be created.

USB Audio application interface - enumerates HID consume transport, HID datalink and USB Audio classes.

USB application for ANC tuning - enumerates USB Audio class.

USB application for Adaptive ANC tuning - enumerates USB Audio class.

◆ UsbApplication_Open()

void UsbApplication_Open ( const usb_app_interface_t *  app)

#include <adk/src/domains/usb/usb_application/usb_application.h>

Open application referenced by a pointer to application interface.

If there is currently active application it is detached and destroyed first. Then the new application is created and attached using application interface callbacks.

If the application provided is already active, function does nothing.

Parameters
apppointer to USB application interface
