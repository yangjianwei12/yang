Detailed Description

A component responsible for controlling voice assistants.

Interface for Voice Assistant clients (VA protocol layers)

Voice Assistant Session state private header.

A component responsible for updating peer device with the active voice assistant.

Message IDs of the messages send by Voice UI service.

GAIA interface with Voice Assistants.

API of the voice UI EQ handling.

Voice UI container private header.

Voice UI configuration header.

API of the voice UI battery handling.

Voice UI audio private header.

Voice UI audio private header with APIs that handle requests that should be done asynchronously (via messages)

API of the voice UI ANC handling.

Responsibilities:

  • Voice Ui control (like A way to start/stop voice capture as a result of some user action) Notifications to be sent to the Application raised by the VA.

The Voice Ui uses Audio Audio domain and Bluetooth BT domain.

Responsibilities:

  • A component responsible for updating the peer device with the active voice assistant

◆ VoiceUi_StartWakeUpWordDetection()

voice_ui_audio_status_t VoiceUi_StartWakeUpWordDetection ( voice_ui_handle_t *  va_handle,
const va_audio_wuw_detection_params_t *  audio_config 
)

#include <adk/src/services/voice_ui/voice_ui/voice_ui_va_client_if.h>

Start Wake-Up-Word detection.

If this function is called when a detection chain is already started and the detection parameters are different, the current chain will be stopped and a new chain will be started with the new parameters.

Parameters
va_handleHandle provided at registration.
audio_configConfiguration related to Wake-Up-Word detection.
Returns
voice_ui_audio_success if a new detection chain is started, voice_ui_audio_already_started if a detection chain with the same parameters is already started. voice_ui_audio_suspended if detection is blocked and will resume later when it is able. voice_ui_audio_not_active if the VA handle is not that of the active assistant. voice_ui_audio_failed if detection fails to start.
