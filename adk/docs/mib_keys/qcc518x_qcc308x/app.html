AdcTemperatureMultiplier

Number (psid) 0x010B 267
Type int32
Default 0x019F0000

Multiplicative factor for aux_adc temperature measurement.

This value is used to multiply the raw temperature reading derived from the hardware in the chip's AUX macro to give a value where differences between readings are in degrees Celsius. The value AdcTemperatureAddition is added to the result to convert to degrees Celsius. This value is a signed 32-bit integer in two's complement representation. It represents a fixed point number found by shifting right by 16 bits. Hence the top 16 bits of the number stored in the MIB represent the integer part of the resulting number while the lower 16 bits a fractional part. For example, 0x000A0000 represents the integer 10 and 0xFFFFC000 represents -1/4. This key must be set for temperature compensation to be performed.

AdcTemperatureAddition

Number (psid) 0x010C 268
Type int16
Default 122

Additive constant for the aux_adc temperature measurement.

This value is added to the result of using the MIB attribute AdcTemperatureMultiplier to give a temperature in degrees Celsius from the chip's AUX macro. The value is 16-bit signed integer: there is no fractional part as the temperature accuracy is not expected to be better than one degree Celsius. The default is zero: this allows a value of AdcTemperatureMultiplier to be set and a reading performed to compare the measured temperature with the known ambient temperature in order to set the correct value of AdcTemperatureAddition. This key must be set for temperature compensation to be performed.

XtalTemperatureLookupTable

Number (psid) 0x010D 269
Type octet_string
Default [d3 ff ba d8 ff c9 dd ff d7 e2 ff e3 e7 ff ec f1 ff fd fb 00 04 05 00 05 0f 00 03 19 ff fd 23 ff f8 2c ff f2 36 ff f0 40 ff f4 4a ff ff 55 00 15]
Allowed Values
Minimum 0
Maximum 48

Lookup table for compensating crystal for temperature

If this MIB key is set, temperature compensation may be performed on the crystal that supplies the system clock. The value gives an array of mappings from temperature to the internal units used for determining the capacitance fine trim applied to the crystal. Each entry consists of 3 octets supplying one 8-bit integer and one 16-bit integer. The first entry is a signed 8-bit integer giving absolute temperature in degrees Celsius. Range for it is -128 to +127. The second entry is a signed 16-bit integer giving delta to the original trim value. Range for it is -32768 to +32767. Outside the range of the minimum and maximum temperature given, no extrapolation is performed: the corresponding delta to original fine trim for the minimum temperature is used for all lower temperatures, and that for the maximum temperature is used for all higher temperatures. Within the range of the temperatures in the table, piecewise linear interpolation is performed between adjacent temperature values to determine the appropriate delta to original fine trim. This key must be set for temperature compensation to be performed.

SiflashEraseRegion0

Number (psid) 0x0117 279
Type uint32
Default 0

Specifies erase region #0 for siflash operations.

Siflash needs to know erase command code and erase block size. Some devices have non-uniform organization, so different commands might be needed for different regions of flash. The value is encoded as follows: MSB 8 bit: erase command code; 8 bit: erase block size exponent (e.g. 16 means 2^16=64KB erase block size). LSB 16 bit: erase region size, in erase blocks, 0xffff means the region spans until the end of device. Erase regions #0...#3 follow one after another and should cover the whole device. If less than 4 erase regions are needed, when those at the end can be left undefined. If all erase regions undefined, erase block parameters are obtained from Curator, otherwise Curator settings are ignored. Example: 0xD8100100 - erase command code 0xD8, block size 10^16 = 64KB, region size is 256 blocks of size 64KB: 16MB.

ChargerTerminationIMONExponentialAverageSpeed

Number (psid) 0x0154 340
Type uint16
Default 32
Allowed Values
Minimum 1
Maximum 256

Moving exponential average speed for IMON ADC readings while charging

During charging IMON ADC readings are taken approximately once per second. The last nine readings are retained at all times, with each new reading replacing the oldest. The median of these nine is then fed into a moving exponential average, again approx once per second. IMON ADC readings start being collected when charging starts or the fast charge current is changed. When nine readings have been taken the first median is generated and that becomes the starting value for the moving exponential average. new_avg = ((median * speed) + (old_avg * (256 - speed))) / 256 median = Median value of the last nine periodic IMON ADC readings old_avg = Previous value of the moving exponential average new_avg = New value of the moving exponential average speed = Value of this MIB key Lower values average more slowly. A value of 1 means 1/256 of each new median would go into the moving exponential average. A value of 256 disables the moving exponential average, using 100% of each new median. External charging uses hardware termination and does not use this key.
