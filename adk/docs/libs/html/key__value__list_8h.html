key_value_key_t  key,
const void *  value,
size_t  size 
)

Add a key-value pair to a key-value list.

This function will add the new key-value pair to the given list unless the key is already in the list or the list is full.

The data passed in via #value is copied to the list and if necessary a new buffer will be allocated and owned by the list to store the copied data. If the buffer cannot be allocated this function will panic.

Small values of 4 octets or less will not require a buffer to be allocated by the list to store the copied data.

Parameters
listKey-value list to add to.
keyKey to insert.
valuePointer to the data to associate with the key.
sizeSize in octets of the data pointed to by value.
Returns
bool TRUE if key-value was added successfully; FALSE otherwise.

Get the value (with an expected size) for a key in a key-value list.

The value is returned as a pointer to the buffer owned by the key-value pair.

◆ KeyValueList_Merge()

bool KeyValueList_Merge ( key_value_list_t  source_list,
key_value_list_t  target_list,
key_value_list_merge_resolve_callback_t  resolve_callback 
)

Merge source key-value list with the target key-value list.

This function will merge all the key value pairs from source list to target list. If same key exists in target list too, then it will compare the value associated with the key. If the value associated are different, then the conflicts needs to be resolved using 'resolve_callback'.

Parameters
source_listSource Key-value list
target_listTarget Key-value list
resolve_callbackCallback function to resolve the conflicts. The callback function can return either KV_LIST_MERGE_CONFLICT_ACTION_ACCEPT_TARGET_LIST or KV_LIST_MERGE_CONFLICT_ACTION_ACCEPT_SOURCE_LIST to resolve the conflict.ie, for accepting target list value or source list value as merged value respectively. If it returns with KV_LIST_MERGE_CONFLICT_ACTION_UNRESOLVABLE, then we will panic. If NULL is passed as resolve_callback, then we will panic if conflicts happen.
Returns
TRUE if success, FALSE otherwise
