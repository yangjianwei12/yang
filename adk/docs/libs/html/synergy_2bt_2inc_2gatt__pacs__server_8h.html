◆ GattPacsServerAddAudioLocation()

bool GattPacsServerAddAudioLocation ( PacsServiceHandleType  handle,
PacsServerDirectionType  direction,
PacsAudioLocationType  audioLocations 
)

This API is used to add Audio Location of SINK/SOURCE PAC A device may choose to have multiple bitmask of PacsAudioLocationType or single PacsAudioLocationType. Application may send complete bitmask by calling this API once or by sending different single PacsAudioLocationType by calling this API multiple times.

This API essentially OR's the single or multiple bit mask with previously set Audio Location of SINK/SOURCE PAC.

For example: If Application wants to set AudioLocation as both Left & Right, it can do so by below ways: a) Application may choose to send 'audioLocations' field as PACS_AUDIO_LOCATION_LEFT | PACS_AUDIO_LOCATION_RIGHT by calling this API once. b) Or application may chose to send 'audioLocation' field as PACS_AUDIO_LOCATION_LEFT first and then by again calling this API with 'audioLocation' fiedl as PACS_AUDIO_LOCATION_RIGHT. In first case (a) there will be single Audio Location change notification send to connected client(s). In second case (b), there will be mulitple Audio Location change notification send to connected client(s), one for Left and another for Left | Right.

Parameters
handleThe server instance handle returned in GattPacsServerInit().
directionDirection specifies SINK/SOURCE Audio location.
audioLocationsDevice wide bitmap of supported Audio Location values for all PAC records where the server supports reception/transmission of audio data
Returns
TRUE if success, FALSE otherwise

◆ GattPacsServerCharChangesDynamically()

void GattPacsServerCharChangesDynamically ( void  )

This API should be exercised carefully so that PACS server does not generate unnecessary notification for the bonded client on re-connection with same char values.

By default if this API is not called, only Available Audio Context notification will be sent to remote client on re-connection as other char values remain unchanged.

This API needs to be called once after GattPacsServerInit() if application finds that char values have changed.

◆ GattPacsServerRemoveAudioLocation()

bool GattPacsServerRemoveAudioLocation ( PacsServiceHandleType  handle,
PacsServerDirectionType  direction,
PacsAudioLocationType  audioLocations 
)

This API is used to remove Audio Location of SINK/SOURCE PAC.

Parameters
handleThe server instance handle returned in GattPacsServerInit().
directionDirection specifies SINK/SOURCE Audio location.
audioLocationsDevice wide bitmap of supported Audio Location values for all PAC records where the server supports reception/transmission of audio data.

NOTE: 1)If Application passes a subset of Device Audio Location to be removed,then subset of audiolocation shall be removed from Device Audio Location set during GattPacsServerAddAudioLocation() API and a notification to the connected client(s) with the new Device Audio Location for SINK/SOURCE PAC shall be triggerred.

2)If Application wants to clear the Device Audio Location for SINK/SOURCE PAC, then it may pass 'audioLocations' field with value PACS_AUDIO_LOCATION_CLEAR. This will remove Device Audio Location set for 'direction' (SINK or SOURCE) passed and will not trigger any notification change to connected client(s). The Application has to then call GattPacsServerAddAudioLocation() with proper bitmask of AudioLocationType to be set for the 'direction' PAC immediately after GattPacsServerRemoveAudioLocation() API.

Returns
TRUE if success, FALSE otherwise

handleThe server instance handle returned in GattPacsServerInit().

This API is used to notify only particular connected client (cid) with Selective Audio Context Availability. Calling this API will not trigger notification for other connected device(s) .

Parameters
handleThe server instance handle returned in GattPacsServerInit().

sinkAudioContextsBitmask of audio data Context Type values for transmission/reception for SNK
sourceAudioContextsBitmask of audio data Context Type values for transmission/reception for SRC
Returns
TRUE if success, FALSE otherwise

NOTE: Once the link with remote device is disconnected then this information is flushed in PACS Server. During subsequent re-connection with the remote device, the application needs to set selective Audio Context Availability again else default Available Audio Contexts shall be notified to remote device.
