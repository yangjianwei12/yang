AVRCP

Overall AVRCP concepts

Naming

Interfaces specific to the target is prefixed with AVRCP_TG or AvrcpTg and interfaces specific to the controller with AVRCP_CT or AvrcpCt

Tasks

Configuration

Before the profile can be used it must be configured by using the configuration interface. During configuration it is chosen which roles to support and the corresponding service records are registered.

(De)activations

In order to accept incomming connections the profile must be activated by using the (de)activation interfaces. Normally only the target should accept incomming connections.

Connection establishment

Refer to the connection interface for details on how to establish and terminate connections.

Security

Refer to the security details

Media Player Registration (TG only)

If the target role is supported at least one Media Player must be registered, which can be done using the Media Player registration interface. When registering a media player all the features as well as notifications and Player Application Settings supported by the player can be specified.

Notifications

The controller can register for notifications in order to be kept up-to-date on the state of the connected target. This is done using the notification interfaces.

Status

The current play status of a target can be retrieved from a controller by using the status interface.

Player Application Settings

In order for a controller to retrieve the Player Application Settings (PAS) supported by a target, the PAS interface can be used. PAS describes which control/menus are available on a target such as equalizer or shuffle controls.

Pass-through commands

The pass-through interface is used for sending simple pass-through commands/responses such as Play, Stop and similar. It is also used for the Group Navigation feature.
