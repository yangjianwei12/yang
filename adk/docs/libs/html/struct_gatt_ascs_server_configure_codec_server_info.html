Detailed Description

The Profile/Application populates this structure and sends it to the ASCS library by calling GattAscsServerConfigureCodecResponse() This structure is used in the response to The fields in this structure are populated by the profile/application and sent to the ASCS library within a GattAscsServerConfigureCodecRsp structure response to an GATT_ASCS_SERVER_CONFIGURE_CODEC_IND.

◆ codecConfiguration

uint8 * codecConfiguration

The ASCS service is responsible for freeing the memory pointed to by 'codecConfiguration'

The ownership of the memory pointed to by this field is dependent on the context in which the GattAscsServerConfigureCodecServerInfo is used. The ownership of this memory is described in the documentation for the following structures:

uint8 retransmissionNumber

Maximum number of times that every CIS Data PDU should be retransmitted. This is a recommendation only which is meant to select between different levels of stream reliability.

All other values: RFU

◆ transportLatencyMax

uint16 transportLatencyMax

Maximum supported transport latency. The maximum time in milliseconds between first transmission of SDU on this ASE, to the end of the last attempt to receive the SDU on the far end of the ASE.

The time range is 5 milliseconds to 4000 milliseconds, and typical values are between 7.5 milliseconds to 90 milliseconds.

Range: 0x0005 to 0x0FA0

All other values: RFU
